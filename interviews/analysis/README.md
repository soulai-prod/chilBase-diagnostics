# Структура анализа интервью — ChillBase AI Audit

## Папки

```
interviews/
├── raw/                          # исходные аудио / PDF / видео
├── transcripts/                  # транскрипты в формате Markdown
│   ├── INT-001-environment.md
│   ├── INT-002-2d.md
│   └── ...
├── data-inventory.json           # реестр всех интервью и их статус
└── analysis/
    ├── by-department/            # глубинный анализ каждого отдела
    │   ├── INT-001-environment-analysis.md
    │   └── ...
    ├── cross-department/         # паттерны поверх всех отделов
    │   ├── shared-pains.md       # боли которые повторяются в нескольких отделах
    │   ├── data-dependencies.md  # данные одного отдела нужные другому
    │   └── ai-maturity-map.md    # карта AI-зрелости по отделам
    └── synthesis/
        ├── ai-opportunity-map.md # Impact-Feasibility Matrix
        ├── ambassador-profiles.md # AI-амбассадоры с классификацией
        ├── deloitte-maturity.md  # профиль AI-зрелости по 7 измерениям
        └── executive-summary.md  # итоговый документ для руководства
```

## Коды для анализа транскриптов

- `[ROUTINE]` — повторяющиеся задачи с чётким паттерном → кандидат на AUTO
- `[PAIN]` — точка потери времени или качества
- `[SCALE_RISK]` — что сломается при росте x2/x5
- `[DATA_ASSET]` — ценные данные, которые уже есть в отделе
- `[AI_SIGNAL]` — упоминание AI, готовность, страх или скепсис
- `[DEPENDENCY]` — критическая зависимость от человека или системы
- `[HIDDEN_PAIN]` — боль, которую лид не назвал явно, но она видна из контекста
- `[CROSS_DEPT]` — зависимость от другого отдела — потенциальное узкое место

## Статусы интервью

- `raw` — транскрипт получен, не обработан
- `coded` — проставлены коды
- `analyzed` — написан анализ по отделу
- `synthesized` — включён в кросс-анализ
